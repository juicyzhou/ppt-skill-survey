# 快速上手

## 1. 先做安全检查

不要直接执行陌生仓库中的安装脚本。安装任何 Skill 前：

1. 阅读 `SKILL.md`、`README`、`LICENSE` 和依赖清单。
2. 查看脚本是否会联网、上传文件、读取环境变量或执行远程代码。
3. 固定 commit/tag，而不是长期追踪不受控的 `main`。
4. 在隔离目录或容器中试用，不要使用真实机密材料。
5. 检查字体、模板、图片和图标的授权。

## 2. 最小可比测试任务

用同一个 5 页 brief 试跑候选项目：

```text
为一家虚构的 B2B SaaS 公司制作 5 页中文季度业务复盘。
受众：管理层；时长：7 分钟；16:9。
页面：封面、关键结论、收入趋势、客户留存、下一季度行动。
数据：收入 120/138/151/173 万元；留存率 91%/92%/90%/94%。
要求：行动标题、数据来源脚注、演讲者备注、无占位符。
交付：主格式 + PDF/图片预览 + 生成源文件。
```

为什么选这个任务：它同时测试中文字体、叙事、图表、数据准确性、备注、可编辑性和多格式输出，规模又足够小。

## 3. 三种最短路径

### 路径 A：原生可编辑 PPTX

1. 选择 Anthropic PPTX、MiniMax PPTX、ppt-master 或 presentation-skill。
2. 明确要求 E3 可编辑、图表数据可修改、保留备注。
3. 先只生成 2 页，打开 PowerPoint 检查对象树和字体。
4. 再生成完整 5 页并渲染接触表。

示例请求：

```text
使用该 PPTX Skill 完成 quickstart brief。所有文本、形状和图表应为原生
PowerPoint 对象；不得把整页栅格化。生成后渲染预览，检查溢出、重叠、
低对比度、占位符和数据一致性，并至少完成一次修复后复验。
```

### 路径 B：HTML-first

Frontend Slides 的上游文档提供 Claude Code 插件和手动复制两种方式。跨 Agent 最稳妥的试用方法是让 Agent 从仓库的 `SKILL.md` 开始，只按需读取引用文件。

示例请求：

```text
使用 Frontend Slides（或选定 HTML Skill）完成 quickstart brief。
输出单文件 16:9 HTML，支持键盘翻页；同时导出 PDF 或逐页 PNG。
不要声称 HTML 源码可编辑等于 PowerPoint 对象可编辑。
```

### 路径 C：image-first

Codex PPT Skill 的核心过程是“大纲和风格 → 逐页生图 → 组装 PPTX”。应先锁定内容，再调用图像模型，避免在昂贵的生图阶段反复改叙事。

示例请求：

```text
使用 Codex PPT Skill 完成 quickstart brief。先提交逐页大纲与视觉规范，
确认内容后再生图。生成完成后对每页做 OCR 和人工数据核对；明确标注最终
PPTX 为图片式，不把它描述成原生可编辑。
```

## 4. 验收清单

### 内容

- [ ] 5 页齐全，行动标题连起来能表达完整故事。
- [ ] 收入与留存数据和 brief 完全一致。
- [ ] 图表坐标、单位、图例、脚注正确。
- [ ] 没有 `TODO`、Lorem ipsum、虚构来源或无法验证的结论。
- [ ] 演讲者备注存在且不是页面正文的简单重复。

### 视觉

- [ ] 16:9，安全边距统一，无裁切、溢出和对象碰撞。
- [ ] 正文在投影环境可读，中文字体存在合理降级。
- [ ] 页面之间有一致系统，也有足够版式变化。
- [ ] 色彩对比、图表标签和数据重点清晰。

### 技术

- [ ] PowerPoint 能打开并保存，无修复警告。
- [ ] 若要求 E3，文字、形状和图表可分别选择与编辑。
- [ ] PDF/PNG 渲染与 PowerPoint 视觉基本一致。
- [ ] 源文件、资产、依赖版本和生成命令已保留。
